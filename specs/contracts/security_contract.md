# Security Contract

## Requirement Set

- `REQ-SEC-001`: Authentication state MUST be validated at trusted ingress boundaries.
- `REQ-SEC-002`: Authorization decisions MUST be explicit and auditable.
- `REQ-SEC-003`: Sensitive fields MUST be redacted from logs and telemetry.
- `REQ-SEC-004`: Security-relevant failures MUST fail closed by default.
- `REQ-SEC-005`: Security policy updates MUST be versioned and traceable.
